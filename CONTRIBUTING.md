# Contributing Atlases

Thanks for sharing your atlas with the What Pressed community!

## Submission Format

Each atlas lives in its own directory under `atlases/`:

```
atlases/
  your-atlas-name/
    atlas.json          # Atlas definition (required)
    thumbnail.png       # 256x256 preview image (required)
    images/             # All image assets (required)
      key-pressed.png
      key-unpressed.png
      ...
```

### atlas.json

Your `atlas.json` must include these fields:

```json
{
  "name": "your-atlas-name",
  "version": 1,
  "semver": "1.0.0",
  "description": "A short description of your atlas",
  "author": "your-name",
  "entries": [ ... ],
  "source_images": []
}
```

- **name** must match the directory name, use lowercase with hyphens (e.g. `wasd-keys`)
- **version** is the internal schema version, keep it at `1`
- **semver** is your atlas release version (semver format: `1.0.0`)
- **description** and **author** are shown in the community browser
- **entries** is the standard What Pressed atlas entry format

### Requirements

- **thumbnail.png** is required - 256x256 PNG recommended
- All images must be PNG, JPG, or WebP
- No image may exceed 2 MB
- Total atlas size must not exceed 20 MB
- Directory name must match `atlas.json` `name` field
- No path traversal (`../`) or absolute paths in any filenames
- No ZIP files, executables, scripts, or other non-image files in `images/`

### What NOT to include

- Don't include copyrighted images you don't have rights to share
- Don't include executable files of any kind
- Don't modify `index.json` - it's auto-generated by CI on merge

## Submitting

1. Fork this repository
2. Create your atlas directory under `atlases/`
3. Open a pull request to `main`
4. CI will validate your submission automatically
5. Once approved and merged, `index.json` is regenerated and your atlas becomes available in What Pressed

## Updating

To update an existing atlas, bump the `semver` field in your `atlas.json` and open a new PR with the changes.
